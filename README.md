## 🧰 Solution Components

This solution is built using multiple integrated components that work together to monitor incoming emails, process email content, generate AI-driven summaries, and return structured output for downstream workflows.

### 1. Email Source (Outlook / Mailbox)

The email system acts as the entry point for the solution. Incoming sales or business emails are monitored through a workflow trigger. Email metadata such as sender, subject, recipients, received time, and HTML body content are captured and passed into the automation pipeline for processing.

**Purpose:**
- Detect incoming email conversations
- Capture raw email content and metadata
- Trigger the summarization workflow

**Data captured:**
- Subject
- Sender
- To recipients
- Received date/time
- Email body (HTML)

---

### 2. Power Automate

Power Automate is used as the orchestration layer for the workflow. It monitors the mailbox, extracts email fields, and sends the required payload to the backend API for AI summarization. Once the summarized response is received, Power Automate can use it in additional business workflows such as sending an email, updating records, or triggering another approval or action flow.

**Purpose:**
- Monitor mailbox for new emails
- Extract email content and metadata
- Send HTTP request to backend service
- Receive structured summary response
- Route output to downstream workflow systems

**Key responsibilities:**
- Email trigger handling
- Payload preparation
- API invocation
- Workflow continuation after summary generation

---

### 3. Google Cloud Function

Google Cloud Function serves as the backend processing layer. It receives the HTTP request from Power Automate, validates the input payload, extracts email details, converts HTML email content into plain text, invokes Gemini for analysis, and returns the final structured HTML summary response. In your code, this is implemented through the `summarize_email` HTTP function. :contentReference[oaicite:0]{index=0}

**Purpose:**
- Expose an HTTP endpoint for Power Automate
- Process incoming email payload
- Convert HTML body to plain text
- Prepare AI prompt
- Call Gemini model
- Return summary in structured HTML format

**Key implementation details:**
- Built using Python
- Uses `functions_framework` for HTTP handling
- Uses Flask request/JSON processing
- Uses BeautifulSoup to parse HTML email body
- Formats timestamps before sending to AI model
- Returns JSON response containing the summary and metadata :contentReference[oaicite:1]{index=1}

**Input handled by function:**
- `email_body`
- `subject`
- `from`
- `receivedDateTime`
- `toRecipients` :contentReference[oaicite:2]{index=2}

---

### 4. Gemini 3 (Vertex AI)

Gemini is the AI engine that performs the actual content analysis and summarization. The Cloud Function initializes Vertex AI, loads the configured Gemini model, and sends a structured prompt containing the email thread in plain text. The prompt instructs Gemini to return a professional HTML summary with sections like context, outstanding needs, message-by-message recap, and condensed summary. In your implementation, the model configured is `gemini-3-pro-preview`. :contentReference[oaicite:3]{index=3}

**Purpose:**
- Understand email thread context
- Identify action items and pending tasks
- Summarize communication history
- Generate structured, professional HTML output

**Summary sections generated by Gemini:**
- Context Summary
- Needs Summary
- Detailed Email-by-Email Summary
- Condensed Recap :contentReference[oaicite:4]{index=4}

**Why Gemini is used:**
- Strong reasoning across long email threads
- Better understanding of conversational context
- Ability to produce structured HTML output
- Suitable for enterprise automation use cases

---

### 5. HTML Email Summary Output

The final response generated by the Cloud Function is returned as structured HTML. This makes the output easy to display directly inside email clients, workflow notifications, or integrated systems. The returned response includes both the generated summary and original email metadata. Your code wraps the Gemini result in a styled HTML container and includes email header details in the output response. :contentReference[oaicite:5]{index=5}

**Purpose:**

- Support direct email rendering

**Benefits:**
- Human-readable
- Ready for downstream automation
- Consistent format across all summarized emails

## 🔄 End-to-End Flow

1. A new email arrives in the monitored mailbox  
2. Power Automate detects the email  
3. Power Automate extracts the subject, sender, recipients, timestamp, and HTML body  
4. Power Automate sends the payload to the Google Cloud Function  
5. The Cloud Function converts HTML to plain text and formats the email content :contentReference[oaicite:6]{index=6}  
6. The Cloud Function sends the prepared prompt to Gemini in Vertex AI :contentReference[oaicite:7]{index=7}  
7. Gemini generates a structured HTML summary  
8. The Cloud Function returns the response as JSON  
9. Power Automate consumes the summary and uses it in email, workflow, or CRM actions

## 📌 Summary

This solution combines email monitoring, workflow automation, cloud-based processing, and generative AI to create an intelligent sales email assistant. Each tool has a clear responsibility:

- **Email** provides the source content
- **Power Automate** manages the workflow
- **Google Cloud Function** handles backend processing
- **Gemini 3** performs summarization and insight generation
- **HTML output** delivers the result in a business-friendly format

Together, these components create an automated, scalable, and enterprise-ready email intelligence workflow.
